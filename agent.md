# 桌面具身智能训练平台 Agent 说明文档

## 1. 项目名称

桌面具身智能训练平台：低成本机器人教学与竞赛套件

## 2. 项目定位

本项目面向高校 AI 社团、机器人兴趣小组、创新创业团队，建设一套低成本、可复用、可教学、可参赛的桌面具身智能实验平台。

它不是单一机器人产品，而是一套完整的社团训练体系，包含：

* 桌面移动小车
* 小型机械臂
* 摄像头视觉模块
* 语音交互模块
* 目标识别与抓取任务
* 路径规划任务
* ROS / Python 教学任务
* Web 控制台
* 仿真环境
* 课程包与实验手册

项目比赛包装名称：

> 面向高校 AI 社团的低成本具身智能实验教学平台

## 3. 项目目标

本项目的核心目标是：用较低成本，让高校社团能够系统训练新人完成从 AI 编程、机器人控制、视觉识别、语音交互到具身智能任务执行的完整学习闭环。

项目需要同时满足三个目标：

1. **社团自用**：可用于纳新、培训、日常教学。
2. **比赛展示**：可包装为创新创业、挑战杯、创新杯项目。
3. **长期迭代**：每年都可以扩展任务、升级硬件、沉淀课程。

## 4. MVP 范围

MVP 不追求高性能机器人，也不追求复杂大模型具身智能。第一阶段只做“能演示、能教学、能复用”的最小闭环。

MVP 必须包含：

1. 一份低成本硬件 BOM。
2. 5 个实验任务。
3. 10 个教学案例。
4. 一个 Web 控制台。
5. 一个仿真环境。
6. 一个社团训练营课程包。
7. 一套可运行的软件工程骨架。
8. 一份面向比赛展示的项目说明。

## 5. 推荐硬件组成

硬件以低成本、易采购、易维护为原则。

### 5.1 移动底盘

可选方案：

* 双轮差速小车底盘
* 四轮麦克纳姆轮小车
* Arduino / ESP32 / Raspberry Pi 控制底盘

MVP 推荐优先使用双轮差速小车，降低控制难度。

### 5.2 机械臂

可选方案：

* 4 自由度桌面机械臂
* 5 自由度小型机械臂
* 舵机机械臂套件

MVP 推荐使用 4 自由度舵机机械臂，能完成简单抓取即可。

### 5.3 视觉模块

可选方案：

* USB 摄像头
* Raspberry Pi Camera
* 手机摄像头推流
* OpenCV 本地视觉识别

MVP 阶段不依赖昂贵深度相机。

### 5.4 语音模块

可选方案：

* 麦克风输入
* 本地关键词识别
* Web 端文字指令模拟语音
* 后续扩展 Whisper / 本地 ASR

MVP 阶段允许用文字指令模拟语音交互，后续再接入真实语音。

### 5.5 控制主机

可选方案：

* 普通笔记本电脑
* Raspberry Pi
* Jetson Nano / Orin Nano

MVP 推荐先使用普通笔记本运行 Web 控制台、视觉识别和仿真环境。

## 6. 软件系统组成

MVP 软件建议分为六层：

1. **Web 控制台**

   * 显示机器人状态
   * 显示任务列表
   * 发送控制指令
   * 展示摄像头画面或模拟画面
   * 展示路径规划结果
   * 展示实验进度

2. **机器人控制服务**

   * 接收控制台指令
   * 管理小车运动
   * 管理机械臂动作
   * 管理任务状态
   * 提供统一 API

3. **仿真环境**

   * 模拟桌面地图
   * 模拟小车坐标
   * 模拟目标物体
   * 模拟抓取任务
   * 支持无硬件演示

4. **视觉识别模块**

   * 支持颜色块识别
   * 支持简单目标检测
   * 支持摄像头输入或模拟输入
   * 输出目标位置

5. **任务编排模块**

   * 路径规划任务
   * 目标识别任务
   * 抓取任务
   * 语音指令任务
   * 综合挑战任务

6. **教学与文档模块**

   * 课程说明
   * 实验手册
   * 教学案例
   * BOM 清单
   * 比赛材料

## 7. MVP 的 5 个实验任务

### 任务 1：机器人基础控制

目标：让新人理解机器人运动控制基础。

内容：

* 前进
* 后退
* 左转
* 右转
* 停止
* 速度调节

验收标准：

* Web 控制台可以发送控制指令。
* 仿真小车可以根据指令移动。
* 后续可替换为真实底盘控制。

### 任务 2：桌面地图路径规划

目标：让新人理解坐标系、地图、路径规划。

内容：

* 创建桌面二维地图
* 设置起点和终点
* 添加障碍物
* 使用 A* 或 BFS 规划路径
* 在 Web 控制台可视化路径

验收标准：

* 用户能在地图上选择起点终点。
* 系统能输出路径。
* 小车能按路径在仿真环境中移动。

### 任务 3：视觉目标识别

目标：让新人理解 OpenCV 和视觉识别流程。

内容：

* 摄像头读取图像
* HSV 颜色识别
* 轮廓检测
* 目标中心点估计
* 输出目标坐标

验收标准：

* 能识别红色、蓝色或绿色目标块。
* 能在控制台显示目标位置。
* 没有真实摄像头时，可以使用测试图片或模拟画面。

### 任务 4：机械臂目标抓取

目标：让新人理解机械臂控制和任务执行。

内容：

* 机械臂角度控制
* 预设动作序列
* 夹爪开合
* 目标抓取
* 放置到指定区域

验收标准：

* 仿真环境中能完成“识别目标—移动到目标—抓取—放置”的流程。
* 真实硬件接入时，可以通过统一接口控制舵机机械臂。

### 任务 5：语音 / 指令驱动综合任务

目标：让新人理解多模态交互和任务编排。

内容：

* 用户输入自然语言或文字指令
* 系统解析意图
* 调用路径规划、视觉识别、机械臂抓取模块
* 完成综合任务

示例指令：

* “移动到 A 点”
* “抓取红色方块”
* “把蓝色物体放到目标区”
* “开始综合挑战任务”

验收标准：

* 控制台支持文字指令输入。
* 系统可以把文字指令映射到具体任务。
* 后续可替换为真实语音识别。

## 8. MVP 的 10 个教学案例

1. Python 基础与机器人控制指令
2. 坐标系、姿态与运动控制
3. Web 控制台与机器人 API 通信
4. OpenCV 图像读取与颜色识别
5. 目标检测与坐标估计
6. BFS / A* 路径规划算法
7. 机械臂舵机角度控制
8. 简单任务状态机设计
9. 文字指令到机器人动作的映射
10. 综合挑战：识别、规划、移动、抓取、放置

## 9. 推荐技术栈

### 9.1 前端

推荐：

* React / Vite / Next.js 任一方案
* TypeScript
* Canvas / SVG 绘制地图
* WebSocket 或 HTTP API 通信

MVP 优先使用实现简单、启动快的方案。

### 9.2 后端

推荐：

* Python
* FastAPI
* Pydantic
* OpenCV
* NumPy

后续可扩展：

* ROS 2
* Gazebo
* MoveIt
* Whisper
* YOLO
* SLAM

### 9.3 仿真

MVP 阶段使用轻量二维仿真即可，不强依赖 Gazebo。

仿真内容包括：

* 二维桌面地图
* 小车位置
* 障碍物
* 目标物体
* 机械臂抓取状态
* 任务执行状态

## 12. 统一接口设计

所有真实硬件和仿真环境都必须通过统一接口调用，避免代码和具体硬件强绑定。

示例接口：

```python
class Chassis:
    def move_forward(self, speed: float): ...
    def move_backward(self, speed: float): ...
    def turn_left(self, speed: float): ...
    def turn_right(self, speed: float): ...
    def stop(self): ...

class Arm:
    def move_to(self, x: float, y: float, z: float): ...
    def open_gripper(self): ...
    def close_gripper(self): ...
    def reset(self): ...

class Vision:
    def detect_objects(self): ...

class Planner:
    def plan_path(self, start, goal, obstacles): ...
```

MVP 允许先实现 Mock / Simulator 版本，后续替换真实硬件驱动。

## 13. 比赛叙事重点

比赛材料需要突出以下几点：

1. **低成本**：相比商用机器人教学平台，成本显著降低。
2. **可复制**：适合多个高校社团部署。
3. **可教学**：不只是硬件，而是完整课程体系。
4. **可竞赛**：能支撑挑战杯、创新杯、互联网+等比赛。
5. **可持续**：社团每年纳新和培训都能使用。
6. **具身智能热点**：贴合 AI 从语言模型走向物理世界的趋势。
7. **软硬件结合**：既有工程实现，也有教育应用场景。

## 14. 非目标

MVP 阶段不要做以下内容：

* 不做昂贵机器人本体。
* 不做复杂仿人机器人。
* 不做无人车级别自动驾驶。
* 不做复杂机械臂动力学仿真。
* 不做过度复杂的 ROS 全家桶。
* 不做大而全的平台商城。
* 不做需要大量经费的硬件采购方案。

## 15. 验收标准

MVP 验收时至少满足：

1. 打开 Web 控制台后，可以看到机器人状态。
2. 可以通过按钮控制仿真小车运动。
3. 可以在地图上设置目标点并生成路径。
4. 可以运行一个视觉识别 Demo。
5. 可以运行一个机械臂抓取模拟任务。
6. 可以输入文字指令触发任务。
7. 文档中包含低成本 BOM。
8. 文档中包含 5 个实验任务。
9. 文档中包含 10 个教学案例。
10. 文档中包含比赛包装说明。

## 16. 开发原则

* 先可运行，再优化。
* 先仿真，再硬件。
* 先教学闭环，再性能提升。
* 先低成本，再高配置。
* 先社团可用，再商业包装。
* 每个模块都要有清晰的教学意义。
* 每个功能都要能服务比赛展示或社团训练。

## 17. 输出要求

每次开发或文档更新完成后，需要说明：

1. 修改了哪些文件。
2. 新增了哪些功能。
3. 如何运行或查看。
4. 当前还缺什么。
5. 下一步建议做什么。

## 18. 项目最终愿景

本项目最终希望成为高校 AI 社团的标准化具身智能训练平台，让新人可以从零开始，在一个低成本桌面环境中完成 AI 编程、视觉识别、运动控制、机械臂抓取、路径规划和综合任务编排训练，并沉淀为可持续建设的社团核心资产。
