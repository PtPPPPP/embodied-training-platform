# 项目答辩大纲（Defense Outline）

> 本文档给出三套材料：
> 1. **11 节完整答辩大纲**（适用于准备 PPT 与问答环节）；
> 2. **3 分钟讲稿提纲**（适用于初赛/快速路演）；
> 3. **5 分钟讲稿提纲**（适用于复赛/决赛）。
>
> 所有"已完成"均与当前 MVP 代码一致；"计划"项明确标注，不夸大。

---

## 一、完整答辩大纲（11 节）

### 1. 开场：为什么做这个项目
- 具身智能是当前 AI 最热的方向之一：从"能说会写"到"能动手做事"；
- 但高校社团新人普遍只会调 API，缺一条通往物理世界的路径；
- 商用平台太贵，社团缺一套能长期复用的训练体系；
- 一句话点题：**我们要让每个想学具身智能的学生，一周内真正动起手来。**

### 2. 痛点：高校 AI 社团缺什么
- 新人不懂 AI 与物理世界的连接；
- 硬件门槛高，劝退新人；
- 教学碎片化，知识点串不起来；
- 项目换届归零，难以沉淀；
- 比赛缺"看得见、动起来"的系统。

### 3. 方案：我们做了什么
- 一套**桌面具身智能训练平台**：低成本套件 + Web 控制台 + 仿真 + 课程体系；
- 核心理念：**先仿真、后硬件；统一接口贯穿始终**；
- 交付的不只是一台机器人，而是"4 周 + 5 实验 + 10 案例"的训练体系。

### 4. 演示：平台如何运行（现场 demo）
- 打开浏览器控制台，展示 10×10 地图、小车、红/蓝目标、障碍；
- 点击按钮控制小车前进/转向，状态实时变化；
- 设起点终点生成避障路径（BFS）；
- 选择红色方块点"模拟抓取"，观察机械臂 `holding` 与目标 `status` 变化；
- 输入"开始综合挑战"，一次完成规划 + 抓取；
- 强调：**全程零硬件，一台笔记本完成。**

### 5. 技术：系统架构与核心模块
- 前端 React + Vite；后端 Python + FastAPI；
- `RobotSimulator` 统一门面，下设 `SimulatedChassis / SimulatedArm / SimpleVision / GridPlanner / TaskManager`；
- 8 个 API 端点覆盖状态/控制/规划/视觉/抓取/指令/复位；
- 强调**统一接口**：仿真后端可逐个替换为真实硬件，上层不动。

### 6. 创新：和普通机器人套件有什么不同
1. 低成本具身智能教学闭环（0 元 → 几千元全梯度）；
2. 仿真与实物统一接口（一套代码两种后端）；
3. 面向社团的课程化平台（不是卖硬件，是给训练体系）；
4. 从 AI 编程到物理执行的完整链路（感知→规划→决策→控制→执行，全程可视）；
5. 可复用的比赛与训练体系（换届不归零）。

### 7. 成本：为什么低成本
- MVP 零硬件：一台笔记本即可；
- 三档 BOM：A（0~100 元）/ B（300~800 元）/ C（1500~3500 元）；
- 关键不是买便宜硬件，而是"统一接口"让同一套代码贯穿三档，避免重复投入。

### 8. 应用：社团、课程、比赛如何使用
- 社团：纳新宣讲 + 4 周训练营；
- 课程：AI/机器人/控制课的大作业与实验；
- 比赛：挑战杯、互联网+、创新杯、大创；
- 跨社团/跨校：可复制的部署方案。

### 9. 进度：MVP 已完成什么（实事求是）
- ✅ FastAPI 后端 + 8 API；React 控制台；二维仿真；
- ✅ 底盘控制、BFS 规划、模拟视觉、模拟抓取、文字指令；
- ✅ 5 示例 + 单测 + 完整文档（总览/BOM/实验/课程/比赛/路线/风险）；
- 📅 计划：真实硬件接入、真实 OpenCV、A*、机械臂放置、展示外壳、多社团试用。

### 10. 规划：后续如何迭代
- 三阶段路线图：MVP（已）→ 实物接入 → 竞赛展示；
- 每阶段留"成果沉淀物"：文档骨架 / 接口验证报告 / 路演材料；
- 远期：ROS 2、YOLO、语音、SLAM、大模型任务规划。

### 11. 结尾：项目价值总结
- 对学生：补上软硬件结合的关键一课；
- 对社团：可持续的核心资产；
- 对老师/学校：可指导、可展示、可申报；
- 对比赛：切中热点、故事完整、看得见动起来；
- 收尾：**低成本 + 可复制 + 可教学 + 可比赛 + 可持续，这就是我们的全部答案。**

## 三、5 分钟答辩讲稿提纲（约 1000 字）

> 适用：复赛、决赛。在 3 分钟版基础上展开技术细节、创新点与价值。

**【0:00–0:40】开场：为什么做**
具身智能让 AI 从屏幕走进物理世界，是当下最重要的人才方向。但普通高校社团被三道墙挡住：硬件太贵、门槛太高、项目换届就归零。我们想拆掉这三道墙。

**【0:40–1:20】痛点**
- 新人会跑模型，却不知道模型输出怎么变成机器人动作；
- 一上来就焊电路、装 ROS，劝退新人；
- 教学碎片化，比赛时只有 PPT 没有实物。

**【1:20–2:10】方案 + demo**
我们的方案是"桌面具身智能训练平台"。现场演示：打开浏览器控制台——10×10 地图、小车、红蓝目标、障碍一目了然。点前进，位置和朝向实时变；设 (0,0) 到 (8,8)，BFS 立刻规划出绕障路径；选红色方块点抓取，机械臂状态变成 holding，目标变成 grabbed；输入"开始综合挑战"，系统自动规划到红色方块并抓取。**全程一台笔记本，零硬件。**

**【2:10–3:00】技术与架构**
前端 React + Vite，后端 Python + FastAPI。核心是 `RobotSimulator` 统一门面，下面五个模块：底盘、机械臂、视觉、规划、任务管理，8 个 API 端点。最关键的工程理念是**统一接口**：现在都是 Simulated 模拟实现，第二阶段接真实硬件时，只换驱动后端，上层任务逻辑、控制台、示例脚本几乎不用改。规划用 BFS，后续升级 A*；视觉现在是模拟目标列表，后续接 OpenCV。

**【3:00–3:50】创新点**
五个创新：低成本全梯度启动、仿真与实物统一接口、面向社团的课程化平台、感知到执行的完整链路、可复用的训练体系。其中"统一接口"和"课程化"是我们和普通机器人套件最大的不同——别人卖硬件，我们给的是一套能跑很多年的训练体系。

**【3:50–4:30】成本与应用**
三档 BOM：A 档 0 到 100 元就是现在的 MVP；B 档 300 到 800 元接真实小车和机械臂；C 档 1500 到 3500 元做决赛路演。社团用来纳新和 4 周训练营，配合 5 个实验和 10 个教学案例；课程当大作业；比赛能打挑战杯、互联网+、创新杯。

**【4:30–5:00】进度与价值**
MVP 已经跑通完整闭环，文档齐备。后续三阶段：接真实硬件、完善课程、做展示与推广。对学生补上软硬件结合的关键一课，对社团是可持续的核心资产，对老师是可指导可申报的载体。**低成本、可复制、可教学、可比赛、可持续——这就是我们的答案。谢谢！**
