# 比赛包装方案（Competition Package）

> 本文档是把项目包装成参赛材料的"主稿"。适用于挑战杯、互联网+、创新杯、
> 大创、SRTP 等赛道。可直接二次加工为申报书、PPT、路演稿。
>
> 风格：务实、可验证、不浮夸。**所有"已完成"均与当前 MVP 代码一致；"计划"项明确标注。**

---

## 1. 项目名称

- 正式名：**桌面具身智能训练平台**
- 比赛包装名：**面向高校 AI 社团的低成本具身智能实验教学平台**

## 2. 一句话介绍

> 一套**零硬件即可启动、几百元即可实物化**的桌面具身智能训练平台，
> 让高校 AI 社团新人在 4 周内走通"感知—规划—控制—执行"的完整闭环，
> 并可长期复用于纳新、教学与竞赛。

## 3. 项目背景

- 具身智能正成为 AI 主战场，产业与学界需求旺盛；
- 高校学生普遍停留在"纯软件 AI"，缺乏软硬件结合经验；
- 商用机器人教学平台昂贵（单台数千~数万元），普通社团难以大规模部署；
- 社团项目换届即归零，缺少可持续沉淀的训练体系。

## 4. 用户痛点

1. 新人只会调 API，不理解 AI 与物理世界的连接；
2. 机器人硬件门槛高，劝退大量新人；
3. 教学任务碎片化，知识点串不起来；
4. 社团项目难以长期沉淀，年年从零开始；
5. 比赛项目缺少"看得见、动起来"的实体/仿真系统。

## 5. 解决方案

- **低成本桌面套件**：三档硬件（A 仿真 / B 实物 / C 展示），从 0 元到几千元全梯度；
- **Web 控制台 + 二维仿真**：浏览器打开即用，**无硬件也能完整演示和教学**；
- **统一接口**：仿真与真实硬件共用一套接口，代码可平滑迁移；
- **课程化训练体系**：4 周课程包 + 5 个实验 + 10 个教学案例；
- **可参赛/可路演**：软硬件结合、全程可视，切中具身智能热点。

## 6. 技术路线

| 层 | 选型 | 说明 |
|----|------|------|
| 前端 | React + Vite + TypeScript | Web 控制台，Canvas/网格渲染地图 |
| 后端 | Python + FastAPI + Pydantic | 机器人控制 API |
| 仿真 | 自研二维栅格仿真（`RobotSimulator`） | 10×10 地图，无 Gazebo 依赖 |
| 规划 | BFS（4 邻域） | 后续可扩展 A* |
| 视觉 | 模拟目标列表（`SimpleVision`） | 后续接 OpenCV / YOLO |
| 执行 | 模拟底盘 + 模拟机械臂 | 后续接真实舵机/电机 |
| 交互 | 文字指令（规则解析） | 后续接 ASR / LLM |

> 技术选型遵循"先可运行、再优化；先仿真、再硬件"（见 `agent.md` 第 16 节）。

## 7. 系统架构

```
Web 控制台（React, :5173）
   │ HTTP
FastAPI 控制服务（:8000）
   │
RobotSimulator（统一门面）
   ├─ SimulatedChassis   前进/后退/左转/右转/停止
   ├─ SimulatedArm       grab / reset
   ├─ SimpleVision       返回模拟目标列表
   ├─ GridPlanner        BFS 路径规划
   └─ TaskManager        文字指令 → 动作
```

8 个 API 端点：`/health`、`/api/robot/state`、`/api/robot/chassis/move`、
`/api/planner/path`、`/api/vision/objects`、`/api/robot/arm/grab`、
`/api/task/command`、`/api/robot/reset`。

## 8. 创新点（重点）

1. **低成本具身智能教学闭环**：0 元仿真起步 → 几百元实物 → 几千元展示，全梯度可启动；
2. **仿真与实物统一接口**：一套代码、两种后端，`Simulated*` 可逐个替换为真实驱动，上层逻辑不动；
3. **面向社团的课程化平台**：交付的不是硬件，而是能跑多年的"4 周 + 5 实验 + 10 案例"训练体系；
4. **从 AI 编程到物理执行的完整链路**：感知（视觉）→ 规划（BFS）→ 决策（指令）→ 控制（底盘）→ 执行（机械臂），全程在控制台可视；
5. **可复用的比赛与训练体系**：每年招新、培训、打比赛复用同一套资产，换届不归零。

## 9. 应用场景

- **社团纳新与培训**：纳新现场演示、4 周新人训练营；
- **课程实践**：AI/机器人/控制相关课程的大作业、实验；
- **竞赛项目**：挑战杯、互联网+、创新杯、大创、SRTP；
- **毕设与科研入门**：具身智能方向的入门载体；
- **跨社团/跨校推广**：可复制的部署方案。

## 10. 商业 / 推广模式

> 本项目以**社团公益 + 教育推广**为主，非纯商业 BP。推广思路：

- **开源 + 文档驱动**：以 README + 课程包降低复制门槛，让其他社团按文档自部署；
- **硬件方案分包**：提供 A/B/C 三档 BOM，社团按预算自选采购；
- **校际联盟**：形成跨校社团共享课程与任务的生态；
- **教师/学院合作**：作为课程实践或教改项目载体，争取经费与学分支持；
- **（远期）教学套件化**：成熟后可考虑与厂商合作推出标准化教学套件。

## 11. 社会价值

- 降低具身智能的学习门槛，让更多普通高校学生有机会接触前沿方向；
- 缩小"纯软件 AI"与"软硬件结合"之间的能力鸿沟；
- 为高校社团提供可持续的科技人才培养载体；
- 推动具身智能教育在高校的普及与公平（低成本 = 可复制）。

## 14. 经费预算

> 详细分档见 [`risk-and-budget.md`](./risk-and-budget.md)。概要：

| 方案 | 预算区间 | 用途 |
|------|----------|------|
| 0 元仿真启动 | 0 ~ 100 元 | 当前 MVP，全员自备电脑 |
| 500 元以内基础 | 300 ~ 800 元 | B 档：真实小车 + 机械臂 |
| 1000~2000 元增强 | 1000 ~ 2000 元 | B 档升级 + 基础展示 |
| 3000 元以上展示 | 1500 ~ 3500 元 | C 档：板载算力 + 外壳 + 路演箱 |

## 15. 风险与对策

> 详细见 [`risk-and-budget.md`](./risk-and-budget.md)。概要：

| 风险 | 对策 |
|------|------|
| 硬件采购风险 | 先仿真后硬件、按需采购、留备件预算 |
| 机械臂精度不足 | MVP 用模拟 + 预设动作；实物用轻物块固定点 |
| 小车控制不稳定 | 仿真先行、限速、加回正逻辑 |
| 新人学习曲线陡 | 4 周分阶课程 + 老带新 + 环境帮扶 |
| 比赛材料包装不足 | 用本方案 + 答辩大纲 + 现场 demo |
| 项目后续维护难 | 统一接口 + 文档沉淀 + 梯队传承 |
| 经费不足 | 0 元方案兜底、分档采购、争取学校支持 |
