# 用户故事（User Stories）

> 用具体角色的视角讲清楚"谁、在什么场景下、用本平台解决什么问题"。
> 配合 [`teacher-support.md`](./teacher-support.md) 一起看，说服力更强。
> 故事均为典型场景，不虚构具体姓名与已发生成果。

---

## 故事 1：社团新人·小林（大一，零基础）

**背景**：小林刚加入 AI 社团，会一点 Python，但从没碰过机器人。

**以前**：社团让他先看一周 ROS 文档，他看了三天就放弃了——"我只是想看看 AI 怎么让东西动起来，为什么要先学这么多？"

**现在（用本平台）**：
- 第 1 周，他在自己笔记本上启动平台，打开控制台，点"前进"，小车真的动了；
- 他用 5 行 Python 调 `/api/robot/chassis/move` 让小车走了一个正方形；
- 第 2 周，他让小车按 BFS 规划的路径绕开障碍走到终点；
- 第 4 周，他输入"开始综合挑战"，系统自动规划并抓起红色方块。

**价值**：小林第一次理解了"AI 怎么连接物理世界"，并且**全程没碰一根线、没焊一个点**。

## 故事 2：社长·老王（大三，带新人很累）

**背景**：老王是社长，每年最头疼的是纳新和培训——讲半天理论，新人听不懂；做硬件 demo，一套设备只能两三个人碰。

**以前**：每年招新靠 PPT，留不住人；培训靠老成员口口相传，换届就失传。

**现在**：
- 纳新现场，他打开控制台当场演示"识别—规划—抓取"，新人直观看到"AI 能动手"；
- 他用现成的 4 周课程包排培训，每周有明确验收，进步看得见；
- 所有内容都在仓库里，**换届不归零**，下届社长直接接着用；
- 不同水平的成员都能找到活干：写算法、搞硬件、做文档。

**价值**：老王有了一套**能复用、能传承、能展示**的核心资产，社团终于不再"年年从零开始"。

---

## 故事 3：指导老师·张老师（想找可指导的项目）

**背景**：张老师想带学生做点具身智能方向的项目，但商用平台太贵，自己搭又没时间。

**以前**：找不到一个低成本、能上手、又能出成果的载体。

**现在**：
- 本平台文档齐全，张老师一眼看懂整体设计与教学意义；
- 学生在 4 周训练营里有清晰交付，**可指导、可验收**；
- 平台可以包装成大创 / SRTP / 课程实践项目，**可申报**；
- demo 现场效果好，**可展示**给学院和评审。

**价值**：张老师拿到了一个**低成本、可指导、可展示、可申报**的项目载体。详见 [`teacher-support.md`](./teacher-support.md)。

---

## 故事 4：比赛评委·李评委（看了太多纯 PPT 项目）

**背景**：李评委每年看几十个参赛项目，多数是"PPT 讲得热闹，实物拿不出来"。

**遇到本项目**：
- 学生现场打开控制台，**看得见、动起来**：小车移动、路径规划、机械臂抓取；
- 故事完整：从痛点（社团缺什么）到方案（统一接口 + 课程体系）到成本（0 元起步），逻辑闭环；
- 切中"具身智能"热点，技术栈清晰（FastAPI + React + 仿真），有真实代码与单测支撑；
- 有明确的后续 roadmap，不是一锤子买卖。

**价值**：李评委看到了一个**真实可演示、逻辑自洽、可持续**的软硬件结合项目。

---

## 故事 5：兄弟社团·隔壁校机器人协会

**背景**：隔壁校机器人协会也想做具身智能入门，但没经费、没课程。

**怎么复用**：
- 按 README 启动，**0 硬件成本**即可复现全部功能；
- 按 BOM 选 B 档（几百元）接入实物；
- 直接复用 4 周课程包与 10 个教学案例；
- 遇到问题查文档和 examples，降低复制门槛。

**价值**：本平台**可复制**，让具身智能教育在更多普通高校落地（低成本 = 可复制 = 教育公平）。

## 用户故事对应的价值主张

| 故事 | 凸显的价值 |
|------|------------|
| 小林（新人） | 可教学、低门槛、软硬件结合 |
| 老王（社长） | 可持续运营、可纳新留人、可传承 |
| 张老师 | 可指导、可展示、可申报 |
| 李评委 | 可比赛、真实可演示、切中热点 |
| 兄弟社团 | 可复制、低成本 |

> 这五条价值（低成本 / 可复制 / 可教学 / 可比赛 / 可持续）正是本项目的核心主张，
