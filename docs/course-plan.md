# 社团训练营课程包（4 周计划）

> 本课程包把 10 个教学案例（[`../lessons/`](../lessons/)）和 5 个实验任务（[`experiments.md`](./experiments.md)）
> 编排成一个 **4 周、每周 2 次集中活动** 的训练营。
>
> 设计原则：**新人零基础可入门，4 周后能独立完成一次综合 demo，并具备参赛基础。**
> 当前 MVP 无需任何真实硬件，全部在仿真 + Web 控制台上完成。

---

## 总览

| 周次 | 主题 | 涉及课程 | 涉及实验 | 里程碑 |
|------|------|----------|----------|--------|
| 第 1 周 | 机器人平台入门 | lesson 01、02、03 | 实验 1 | 能控制仿真小车 + 看懂控制台 |
| 第 2 周 | 视觉识别与地图规划 | lesson 04、05、06 | 实验 2、3 | 能获取目标坐标 + 规划避障路径 |
| 第 3 周 | 机械臂控制与任务编排 | lesson 07、08、09 | 实验 4 | 能抓取 + 理解任务编排 |
| 第 4 周 | 综合挑战与比赛展示 | lesson 10 | 实验 5 | 完成综合 demo + 比赛材料 |

**每周结构固定**：学习目标 → 课程安排 → 实验任务 → 交付物 → 考核方式。

---

## 准备工作（第 0 周 / 开营前）

- 每位成员自备一台能装 Python 的电脑（A 档，0 硬件成本）；
- 老成员提前跑通：后端 `python -m uvicorn main:app --app-dir backend --reload`、前端 `npm run dev`；
- 准备一份"环境安装避坑清单"（Python 版本、依赖、端口、CORS），开营时一对一帮扶；
- 确认 `http://127.0.0.1:5173` 能正常显示控制台、`/health` 返回 ok。

> 原则：**绝不让新人卡在"装不上环境"。** 环境问题由助教集中解决。

## 第 1 周：机器人平台入门

### 学习目标
- 能搭好开发环境，打开 Web 控制台；
- 理解"一次 API 调用如何变成机器人的一次动作"；
- 掌握用 Python 控制机器人基础动作；
- 理解坐标系、朝向、运动控制的基本概念。

### 课程安排（2 次集中活动，每次约 2 小时）

**第 1 次活动：环境与平台认知**
- 开营介绍：项目是什么、为什么做、4 周能学到什么（30 min）
- 环境安装：Python + 后端 + 前端，打开 Web 控制台（40 min）
- [lesson-01](../lessons/lesson-01-python-control.md)：Python 基础与机器人控制指令（动手，50 min）

**第 2 次活动：运动控制与坐标**
- [lesson-02](../lessons/lesson-02-robot-motion.md)：坐标系、姿态与运动控制（讲授 + 动手，60 min）
- [lesson-03](../lessons/lesson-03-web-console.md)：Web 控制台与机器人 API 通信（动手，40 min）
- [实验 1](./experiments.md#实验-1机器人基础控制)：机器人基础控制（验收，20 min）

### 实验任务
- **实验 1：机器人基础控制**（`POST /api/robot/chassis/move`，前进/后退/转向/停止/调速）

### 交付物
- 每人能独立启动平台、打开控制台；
- 提交一段控制脚本或控制台截图（让小车前进、转向）；
- 填写第 1 周学习记录（理解了什么、卡在哪里）。

### 考核方式
- 能否独立启动平台并控制小车（实验 1 验收标准）；
- 加分项：实现"走正方形"轨迹、键盘实时控制。

## 第 2 周：视觉识别与地图规划

### 学习目标
- 理解视觉识别在系统中的角色（把场景变成带坐标的目标列表）；
- 能调用视觉 API 获取目标坐标，并理解"模拟视觉"与真实 OpenCV 的关系；
- 理解栅格地图，能用 BFS 做避障路径规划；
- 把"感知"和"规划"两条线接起来。

### 课程安排（2 次集中活动）

**第 3 次活动：视觉识别**
- [lesson-04](../lessons/lesson-04-opencv-color-detection.md)：图像读取与颜色识别（含 OpenCV 概念，60 min）
- [lesson-05](../lessons/lesson-05-target-localization.md)：目标检测与坐标估计（动手，40 min）
- [实验 3](./experiments.md#实验-3视觉目标识别)：视觉目标识别（验收，20 min）

**第 4 次活动：路径规划**
- [lesson-06](../lessons/lesson-06-path-planning.md)：BFS / A* 路径规划算法（讲授 + 动手，60 min）
- [实验 2](./experiments.md#实验-2桌面地图路径规划)：桌面地图路径规划（动手 + 验收，60 min）

### 实验任务
- **实验 2：桌面地图路径规划**（`POST /api/planner/path`，BFS）
- **实验 3：视觉目标识别**（`GET /api/vision/objects`，`SimpleVision`）

### 交付物
- 提交视觉识别结果截图（目标列表 + 坐标）；
- 提交一张带障碍的地图及规划出的路径截图；
- 把"识别到的目标坐标"作为路径终点，跑一次联动。

### 考核方式
- 能获取目标列表并解读坐标（实验 3 验收）；
- 能在地图上规划出无碰撞路径（实验 2 验收）；
- 加分项：实现 A*、路径平滑、多颜色识别。

## 第 3 周：机械臂控制与任务编排

### 学习目标
- 理解机械臂控制（`grab` / `reset`）与状态变化；
- 理解任务状态机，能把多个动作组织成稳定流程；
- 能把文字指令映射为机器人动作（规则解析）。

### 课程安排（2 次集中活动）

**第 5 次活动：机械臂控制**
- [lesson-07](../lessons/lesson-07-robot-arm-control.md)：机械臂舵机角度控制（讲授 + 动手，60 min）
- [实验 4](./experiments.md#实验-4机械臂目标抓取)：机械臂目标抓取（动手 + 验收，60 min）

**第 6 次活动：任务编排与指令**
- [lesson-08](../lessons/lesson-08-task-state-machine.md)：简单任务状态机设计（讲授 + 动手，60 min）
- [lesson-09](../lessons/lesson-09-command-understanding.md)：文字指令到机器人动作的映射（动手，40 min）
- 把抓取流程接入状态机（20 min）

### 实验任务
- **实验 4：机械臂目标抓取**（`POST /api/robot/arm/grab`、`/api/robot/reset`）

### 交付物
- 提交抓取—复位的操作记录（状态变化截图）；
- 提交一个可运行的任务状态机思路（伪代码或流程图）；
- 提交至少 3 条可被解析的文字指令及其 `intent`。

### 考核方式
- 能抓取红/蓝方块并观察状态变化（实验 4 验收）；
- 能讲清 `TaskManager` 如何把文字映射为动作；
- 加分项：抓取真实目标（B 档）、组合指令解析。

## 第 4 周：综合挑战与比赛展示

### 学习目标
- 把前 3 周的"控制/规划/感知/执行"收口成一个综合任务；
- 能独立调试一个完整 demo；
- 理解项目如何用于比赛展示，能参与汇报与材料准备。

### 课程安排（2 次集中活动）

**第 7 次活动：综合集成**
- [lesson-10](../lessons/lesson-10-final-challenge.md)：综合挑战（识别、规划、移动、抓取）（动手，90 min）
- [实验 5](./experiments.md#实验-5文字指令综合任务)：文字指令综合任务（动手 + 验收，30 min）
- Demo 调试：每组跑通一次完整综合任务（自由调试）

**第 8 次活动：展示与汇报**
- 结营 Demo Day：每组演示综合任务（60 min）
- 项目汇报：讲清楚做了什么、怎么做的、价值在哪（30 min）
- 比赛材料准备：分工对接宣传视频/海报/答辩稿（30 min）

### 实验任务
- **实验 5：文字指令综合任务**（`POST /api/task/command`，含"综合挑战"）

### 交付物
- 每组一个可现场演示的综合 demo（识别→规划→移动→抓取）；
- 一份小组项目汇报（5 分钟版）；
- 协助产出比赛材料初稿（对接 [`competition-package.md`](./competition-package.md)）。

### 考核方式
- **结业 demo**：能否现场跑通综合任务（实验 5 验收）；
- **汇报表达**：能否在 3~5 分钟讲清项目；
- 颁发"训练营结业"记录，优秀者进入比赛/核心组。

## 训练营运营建议

### 分组与角色
- 2~4 人一组，角色建议：**算法**（写规划/视觉）、**硬件**（B 档接实物）、**文档/展示**（写汇报、做材料）；
- 不同角色都能找到参与点，避免"只有写代码的人在干活"。

### 节奏控制
- 每次活动**先讲 30%、动手 70%**，杜绝全程念 PPT；
- 每周留 1 个"验收里程碑"，让进步看得见；
- 环境问题集中帮扶，绝不让新人卡在"装不上环境"。

### 老带新
- 老成员担任"助教"，每组配 1 名，负责答疑和验收；
- 老成员同时推进 [`roadmap.md`](./roadmap.md) 第二/三阶段（实物、比赛），形成梯队。

### 沉淀与复用
- 每期训练营结束后，把优秀脚本、踩坑记录归档；
- 课程包本身就是社团核心资产，**逐年迭代而非推翻重来**。
