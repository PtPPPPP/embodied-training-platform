# 项目总览（Project Overview）

> **桌面具身智能训练平台**：低成本机器人教学与竞赛套件
>
> 比赛包装名称：**面向高校 AI 社团的低成本具身智能实验教学平台**

本文是整个项目的"导读页"。如果你只能读一篇文档，就读这一篇。
BOM、实验任务、课程包、比赛材料、路线图都围绕本文的范围展开。

---

## 0. 一句话定位

用 **一台笔记本电脑 + 一套低成本桌面套件**，让高校 AI 社团的新人
在 4 周内走通"从 AI 编程到物理执行"的完整闭环：
看得到、控得住、抓得起、讲得清、能比赛、可复用。

**当前 MVP 已经可以在没有任何真实硬件的情况下完整演示和教学**——
一台装了 Python 的普通电脑就够了。

---

## 1. 项目背景

### 1.1 具身智能（Embodied AI）正在成为 AI 的主战场
AI 的焦点正从"能说会写"转向"能动手做事"。大模型让机器听懂了人话，
但真正走向物理世界——移动、避障、抓取、协作——
需要把感知、决策、控制、执行串成一条链路。
**高校学生如果只停留在"调 API / 训模型"，会错过这一波最重要的人才机会。**

### 1.2 高校学生面临"纯软件 AI"的天花板
社团里很常见的现象：新人会写 Python、会跑模型，但不知道
模型输出的数字怎么变成机器人的一个动作、
一张摄像头画面怎么变成机器人能用的坐标、
"抓起红色方块放到 A 点"这样一句话怎么拆成可执行步骤。
**软件 AI 和物理世界之间，有一道社团新人很难独立跨越的鸿沟。**

### 1.3 商用机器人教学平台太贵
市面成熟的机器人教学平台单价往往数千到数万元，配齐一个实验室动辄十几万。
对普通高校社团来说：买不起、买得起也轮不过来、老师一换就闲置。
**社团需要的是能"低成本复制、人手可上手、长期可复用"的训练平台。**

### 1.4 社团需要可持续沉淀的训练体系
很多社团的现状是：每年招新、做几个 demo，换届归零，下一年又从零开始。
本项目要解决的，正是"年年从零开始"的问题。

## 3. 解决方案

本平台不是"一个机器人"，而是"一套社团训练体系"，由六个部分组成：

1. **低成本桌面机器人套件**：双轮差速小车 + 4 自由度舵机机械臂 + 摄像头；
   三档硬件方案（A 仿真优先 / B 基础实物 / C 增强竞赛），从 **0 元到几千元**全梯度覆盖。
2. **Web 控制台**：浏览器打开即用，显示状态、发指令、可视化地图与路径。
3. **仿真环境**：二维栅格仿真，地图/小车/障碍/目标/抓取状态全部可视化，**无硬件也能完整演示**。
4. **核心能力模块**：视觉识别、路径规划、机械臂抓取、文字指令解析。
5. **配套训练营课程 + 实验任务**：4 周课程包 + 5 个实验 + 10 个教学案例。
6. **统一接口**：仿真与真实硬件共用同一套接口，**仿真里写好的代码，换了后端几乎不用改**。

> **关键：当前 MVP 已用纯仿真跑通完整闭环。** 没有经费也能立即开始全部教学。

## 6. MVP 范围（实事求是）

MVP 不追求高性能机器人，**只做"能演示、能教学、能复用"的最小闭环，且完全不依赖真实硬件。**

### MVP 已实现（与代码一致）
- 后端机器人控制 API（FastAPI）；
- Web 控制台（React + Vite）；
- 二维桌面地图（**10×10 栅格**）；
- 小车前进、后退、左转、右转、停止；
- **BFS** 路径规划（4 邻域，可设障碍）；
- 模拟视觉目标列表（红/蓝方块）；
- 模拟机械臂抓取（`grab`）与复位（`reset`）；
- 文字指令触发任务（规则解析）；
- 5 个示例脚本 + 基础项目文档。

### MVP 明确不做（非目标，列入 [`roadmap.md`](./roadmap.md) 后续）
- 不接真实小车 / 机械臂 / 摄像头 / ROS / 云服务 / 大模型；
- 不做复杂动力学仿真、SLAM、多机器人协作；
- 不做需要大量经费的硬件采购。

## 7. 系统架构（与代码一致）

```
┌──────────────────────────────────────────────┐
│  Web 控制台（React + Vite，端口 5173）          │
│  状态面板 / 小车控制 / 路径规划 / 机械臂 / 指令 / 地图 / 日志
└───────────────────┬──────────────────────────┘
                    │  HTTP（fetch）
┌───────────────────▼──────────────────────────┐
│  FastAPI 机器人控制服务（端口 8000）            │
│  /api/robot/*  /api/planner/*  /api/vision/*  /api/task/*
└───────────────────┬──────────────────────────┘
                    │
┌───────────────────▼──────────────────────────┐
│  RobotSimulator（10×10 仿真核心，统一门面）     │
│   ├─ SimulatedChassis   底盘：前进/后退/左转/右转/停止
│   ├─ SimulatedArm       机械臂：grab / reset
│   ├─ SimpleVision       视觉：返回模拟目标列表
│   ├─ GridPlanner        规划：BFS 4 邻域
│   └─ TaskManager        任务：文字指令 → 动作
└──────────────────────────────────────────────┘
```

**统一接口理念**：所有真实硬件和仿真都通过统一接口调用。
当前 MVP 的 `Simulated*` 实现，后续可逐个替换为真实硬件驱动，
而上层的任务逻辑、Web 控制台、示例脚本**几乎不用改**。

> 目标统一接口定义见 `agent.md` 第 12 节；当前 MVP 实现的精简版见各 `backend/robot/*.py`。

---

## 8. 核心创新点（比赛重点）

1. **低成本具身智能教学闭环**：从 0 元仿真到几千元实物，全梯度可启动；
2. **仿真与实物统一接口**：一套代码、两种后端，教学和实物无缝衔接；
3. **面向社团的课程化平台**：不是卖硬件，而是给一套能跑多年的训练体系；
4. **从 AI 编程到物理执行的完整链路**：感知→规划→决策→控制→执行，全程可视；
5. **可复用的比赛与训练体系**：每年招新、培训、打比赛都能用同一套资产沉淀。

## 11. 结语

我们不是要做一台"更便宜的机器人"，
而是要让 **每一个想学具身智能的高校学生，都能在一周内真正动起手来**。
低成本，是为了让更多人有机会；可持续，是为了让社团年年站在前人肩膀上。
