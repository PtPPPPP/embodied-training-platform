# 低成本硬件 BOM（物料清单）

> 核心原则：**没有经费也能开始，有经费再逐步升级。**
>
> ⚠️ **当前 MVP 完全不依赖真实硬件**——一台装了 Python 的普通电脑即可跑通全部功能。
> 本 BOM 描述的是"如果要做实物 / 上比赛，需要买什么"，属于 [`roadmap.md`](./roadmap.md) 第二、三阶段的采购参考。
>
> 所有价格为 **人民币合理区间**（2026 年市价参考），非具体电商报价，采购以实际渠道为准。本文不推荐具体供应商。

---

## 0. 方案总览

| 档位 | 名称 | 目标 | 一套大致成本 | 状态 |
|------|------|------|--------------|------|
| **A** | 极低成本仿真优先版 | 没有经费也能开始 | **0 ~ 100 元**（用现有电脑） | ✅ **MVP 当前即此档，无需采购** |
| **B** | 基础实物教学版 | 真实小车 + 简单机械臂 | **约 300 ~ 800 元** | 📅 第二阶段 |
| **C** | 增强竞赛展示版 | 提高展示效果与扩展性 | **约 1500 ~ 3500 元** | 📅 第三阶段 |

> 三档**递进**，不是并列。**选型口诀：先仿真（A）→ 再实物（B）→ 最后展示（C）。**
> A 档对应软件当前状态；B 档让代码走进物理世界；C 档为决赛路演加分。

---

## 1. A 档：极低成本仿真优先版（0 ~ 100 元）

### 目标
没有经费也能立即启动，完成全部软件教学和仿真演示。
**这就是当前 MVP 的运行档位。适合第一周启动、纳新宣讲、无硬件教学。**

### 包含内容

| 名称 | 用途 | 是否 MVP 必需 | 预计价格区间 | 可替代方案 | 采购注意 |
|------|------|----------------|--------------|------------|----------|
| 普通电脑（笔记本/台式） | 跑 FastAPI 后端 + React 前端 + 仿真 | ✅ 必需 | 社员自带，0 元 | 任意能跑 Python 的电脑 | 4GB 内存起步，无需独显 |
| Python 3.10+ 环境 | 运行后端与示例 | ✅ 必需 | 0 元（开源） | venv / Anaconda | 统一 Python 版本，避免环境冲突 |
| Node.js（前端） | 运行 Web 控制台 | ✅ 必需 | 0 元（开源） | LTS 版本即可 | `npm install` 装依赖 |
| 浏览器 | 打开 Web 控制台 | ✅ 必需 | 0 元 | Chrome / Edge / Firefox | 现代浏览器即可 |
| 二维仿真环境（项目自带） | 模拟 10×10 地图/小车/目标/抓取 | ✅ 必需 | 0 元 | `RobotSimulator` | MVP 核心交付物，已实现 |
| USB 摄像头（可选） | 真实图像识别练习 | ⬜ 可选 | 30 ~ 80 元 | 手机摄像头推流 | 有则更好，没有用模拟目标列表 |
| 纸质/打印桌面地图 | 把仿真地图变成实物参照 | ⬜ 可选 | 5 ~ 20 元 | A3 打印 / 白板手绘 | 10×10 格子，便于讲解坐标 |
| 彩色方块道具 | 视觉与抓取目标参照 | ⬜ 可选 | 5 ~ 15 元 | 积木 / 折纸 / 便签 | 红/蓝两色（对应仿真里的 red_cube / blue_cube） |

- 仿真里的目标对应：`red_cube`（红色方块，坐标 6,2）、`blue_cube`（蓝色方块，坐标 8,7）。

## 2. B 档：基础实物教学版（约 300 ~ 800 元）

### 目标
实现**真实小车 + 简单机械臂**，让代码从仿真走向物理世界。
**适合社团进入第二阶段、准备比赛展示的核心 demo。**

## 4. 横向对比与采购建议

### 4.1 三档对比

```
成本   A ●─────────────────  0~100 元   （仿真，当前状态）
       B ●●●●─────────────  300~800 元  （实物，第二阶段）
       C ●●●●●●●●●●●──────  1500~3500  （展示，第三阶段）

能力   A  软件教学 + 仿真演示（已实现）
       B  真实小车 + 机械臂抓取（比赛主力）
       C  全向移动 + 边缘AI + 语音 + 外壳（路演）

时机   第1周启动（已满足）  →  社团第二阶段  →  比赛决赛/路演
```

### 4.2 采购原则
1. **先 A 后 B 后 C**：不要一上来买 C 档；
2. **按需采购，能仿真就不买**：MVP 全部用仿真完成；
3. **选社团熟悉的生态**：主控板选大家用得顺手的；
4. **统一接口优先于具体型号**：硬件随时可换，接口设计要稳定；
5. **安全第一**：电池/电源注意电压与发热；
6. **留出耗材与备件预算**：螺丝、杜邦线、备用舵机/电机易损耗；
7. **保留发票/凭证**：便于学校报销和比赛经费核销。

### 4.3 成本控制小技巧
- 摄像头用手机推流代替，省 30~80 元；
- 地图用打印拼接代替定制板；
- 目标块用彩色积木/折纸代替专用道具；
- 主控板优先 ESP32；
- 能用社团现有耗材就不重复买。
