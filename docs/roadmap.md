# 项目路线图（Roadmap）

> 本路线图分三个阶段。**每个阶段都有明确的目标和退出条件（验收点）**，
> 上一阶段不闭环，不盲目进入下一阶段。
>
> 时间为相对周期（周/月），具体节奏按社团实际情况调整。

---

## 总览

```
第一阶段：MVP 阶段            第二阶段：实物接入阶段       第三阶段：竞赛展示阶段
（能演示 / 能教学 / 能纳新）  →（真车 / 真臂 / 真摄像头）  →（可参赛 / 可推广 / 可路演）
   仿真 + Web + 文档闭环          硬件接入 + 统一接口验证       展示外壳 + 课程完善 + 多社团试用
```

---

## 第一阶段：MVP 阶段

### 目标
**能演示、能教学、能用于纳新。**
此阶段不依赖任何真实硬件，全部用仿真 + Web 控制台跑通完整闭环。

### 内容

| 模块 | 具体内容 | 状态定位 |
|------|----------|----------|
| 仿真环境 | 二维桌面地图、小车、障碍物、目标、抓取状态可视化 | ✅ 本阶段核心交付 |
| Web 控制台 | 状态显示、指令下发、路径与摄像头可视化 | ✅ 本阶段核心交付 |
| 5 个实验任务 | 基础控制 / 路径规划 / 视觉识别 / 机械臂抓取 / 综合指令 | ✅ 本阶段核心交付 |
| 10 个教学案例 | lesson-01 ~ lesson-10 | ✅ 本阶段核心交付 |
| 基础 BOM | 三档方案（A/B/C） | ✅ 本阶段核心交付 |
| 统一接口骨架 | `Chassis / Arm / Vision / Planner` + Mock/Simulator 实现 | ✅ 本阶段核心交付 |

### 退出条件（验收点）
- [ ] 打开 Web 控制台能看到机器人状态；
- [ ] 能用按钮控制仿真小车前进/后退/转向/停止；
- [ ] 能在地图上设起点终点并生成路径；
- [ ] 能跑一个视觉识别 demo（颜色块）；
- [ ] 能跑一个机械臂抓取模拟任务；
- [ ] 能输入文字指令触发综合任务；
- [ ] 文档含 BOM / 5 实验 / 10 案例 / 比赛材料。

### 价值
- 纳新宣讲时**当场演示**，无需带硬件；
- 新人第一周就能上手全部课程；
- 形成可复用的软件骨架，为第二阶段硬件接入打好地基。

---

## 第二阶段：实物接入阶段

### 目标
**接入真实小车、机械臂、摄像头，验证"仿真与实物统一接口"的承诺。**
让代码从仿真走进物理世界。

### 内容

| 模块 | 具体内容 |
|------|----------|
| 真实底盘控制 | 双轮差速小车 + ESP32/Arduino/Pico，实现 `Chassis` 实物后端 |
| 真实机械臂控制 | 4 自由度舵机臂，实现 `Arm` 实物后端（角度控制 + 预设动作） |
| 摄像头识别 | USB 摄像头 + OpenCV，实现 `Vision` 实物后端 |
| 桌面地图实物演示 | 在真实桌面地图板上跑通"识别—移动—抓取—放置" |
| 社团训练营试运行 | 用完整 4 周课程包带一届新人，收集反馈 |
| 接口一致性验证 | 仿真后端与实物后端跑同一套任务代码，结果可对齐 |

### 退出条件（验收点）
- [ ] 仿真里跑通的任务，**几乎不改逻辑**就能驱动真实硬件；
- [ ] 真实小车能按规划路径移动到目标点；
- [ ] 真实机械臂能完成抓取—放置；
- [ ] 真实摄像头能识别彩色目标并输出坐标；
- [ ] 完成一届新人训练营试运行并产出改进清单。

### 风险提示（详见 [`risk-and-budget.md`](./risk-and-budget.md)）
- 机械臂精度不足 → MVP 用轻物块 + 预设动作规避；
- 小车控制不稳定 → 仿真先行、限速、加回正逻辑；
- 硬件一致性差 → 统一接口隔离差异。

### 价值
- 比赛答辩时"看得见、动起来"，说服力上一个台阶；
- 验证"统一接口"这一核心创新点不是口号，而是可复现的工程事实；
- 训练营试运行沉淀出真实的教学反馈。

## 第三阶段：竞赛展示阶段

### 目标
**形成可参赛、可推广、可路演的完整项目。**
从"社团自用"走向"多社团可复制 + 对外可展示"。

### 内容

| 模块 | 具体内容 |
|------|----------|
| 展示外壳 | 亚克力/3D 打印外壳，让系统像"一个产品" |
| 课程体系完善 | 根据试运行反馈迭代课程包与实验手册 |
| 比赛路演材料 | 宣传视频、海报、答辩 PPT、Demo 脚本 |
| 多社团试用 | 推广到 2~3 个兄弟社团 / 兄弟院校试用 |
| 指导老师反馈 | 邀请老师参与并收集改进建议 |
| 经费预算与推广方案 | 形成可申报、可复制的推广与预算材料 |

### 退出条件（验收点）
- [ ] 至少参加一次正式比赛（挑战杯 / 互联网+ / 创新杯等）；
- [ ] 至少 1 个外部社团/小组完成试用并反馈；
- [ ] 完成一套对外可交付的课程包 + Demo 视频；
- [ ] 形成"其他社团如何复制本平台"的部署指南。

### 价值
- 形成学校层面的科技创新成果与宣传素材；
- 把"一个社团的项目"升级为"可推广的训练平台";
- 为后续申报大创 / SRTP / 教改项目打下基础。

## 第四阶段及以后（远期展望，非承诺）

> 以下为远期方向，**不是当前承诺**，仅作为长期愿景参考。

- **ROS 2 接入**：对接更主流的机器人生态（Gazebo / MoveIt）；
- **更强感知**：YOLO 目标检测、深度摄像头三维感知；
- **真实语音**：接入 Whisper / 本地 ASR，实现语音指令；
- **SLAM 建图**：让小车具备自主建图与导航能力；
- **多机器人协作**：多车协同任务；
- **大模型任务规划**：用 LLM 把自然语言拆解为机器人任务序列；
- **平台化运营**：形成跨校社团联盟，共享课程与任务。

这些方向与 `agent.md` 第 11 节"P2 后续扩展"一致。

## 阶段依赖与节奏建议

```
[第一阶段 MVP]  ──必须闭环──▶  [第二阶段 实物]  ──必须闭环──▶  [第三阶段 比赛]
   仿真/Web/文档                  硬件接入/接口验证             展示/推广/路演
      │                              │                             │
      └ 纳新可用                     └ 比赛主力 demo                └ 对外推广
```

- **不要跳级**：第一阶段仿真闭环没做好，直接上硬件会陷入"调硬件"泥潭；
- **每阶段留 1 个"成果沉淀物"**：MVP 留文档与骨架，实物留接口验证报告，比赛留路演材料；
- **每年一届新人 = 一次路线图循环**：新成员从第一阶段重新走一遍，老成员推进后续阶段。

---

## 一句话总结

> **先让它跑起来（MVP），再让它动起来（实物），最后让它亮起来（比赛）。**
> 每一步都留下可复用的资产，让社团年年站在前人肩膀上。
